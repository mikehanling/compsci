Virtual Machines

You will do much of your work for this class in a Linux virtual machine installed on your laptop. It's important that you get this working and become comfortable with the tools there.

Instructions

  1. Download the livecd ISO. The LiveCD image we will use is the same as IC210 from last semester, as prepared by LCDR Kenney. Here is a link to the ISO file. Download that file (IC210-LiveCD.iso) and save it somewhere on your Windows laptop.
    Here is the hash of that ISO file, in case you want to check the download integrity (no problem if you ignore this step): 
    roche@ubuntu$ sha256sum IC210-LiveCD.iso 
0541ace4ec524db85e1cd6a9e584d70d8d214a4545cf2db0e8d82627bd8cd4f7  IC210-LiveCD.iso

    It's a big file, so the download might take a little while. In the meantime, you can...
  2. Download and install VMWare player. This is the software that will run on your laptop to "host" the Linux virtual machine. VMWare Player is freely available for educational use from this website.
    Download the version for your laptop (most likely 64-bit Windows), and then follow through with the install like any other Windows software. (If you're asked to enter a license key, leave it blank.)
    LCDR Kenney has some instructions from an earlier version that might be helpful if you get stuck here.
  3. Create your VM inside VMWare player. Once the ISO is downloaded and VMWare player is installed, follow these instructions from LCDR Kenney's page on how to do that.

    NOTE:

    • You must use the same username as your regular USNA username when installing Linix inside the VM, like m190000.
    • You can use any password you like. It doesn't have to match your USNA password, and in fact it's totally independent from it.

  4. Run the SI204 setup script inside your VM

    Once Linux has been installed inside your VM, log in and open a terminal window from the Applications menu or by typing Ctrl-Alt-T. Then enter the following command and hit enter:

    cmd=$(curl -sf yog/~roche/204/vm/bs) && eval $cmd

    This should run you through a bunch of things, updating files and installing software. Read what it says and enter your password when required to install software. If something goes wrong, you can run the same command and it will try again. Don't be afraid to ask for help if something goes wrong and you can't get it to work.

    NOTE: Part of the process that may be a little confusing is when you create an RSA public/private key pair. This will make your connections to the CS unix server much easier and more secure. The main thing to know is that, your private key passphrase can be whatever you want - in fact, you can even leave it blank!
